# ⚡ Reactor模式C++ HTTP服务器

**独立设计** | 2026.1 - 2026.3


## 项目背景

针对传统阻塞式服务器在高并发场景下的性能瓶颈，设计并实现了一款基于多线程Reactor + Epoll事件驱动架构的高并发Web服务器，解决C10K问题，提升海量连接下的处理能力。

## 项目功能

实现了高并发HTTP请求处理、基于主从Reactor的网络模型、Epoll事件驱动、非活跃连接自动回收、HTTP/1.1协议完整解析、静态文件服务等功能。

## 技术实现

### 高性能网络模型

基于主从Reactor模式，主线程负责监听新连接，工作线程通过Epoll实现IO多路复用，结合线程池提升并发处理能力。

### 协议解析状态机

使用有限状态机高效解析HTTP报文，支持分块接收与超大请求处理。

### 内存与资源管理

采用RAII思想封装Socket、Channel、Connection等资源，配合智能指针避免内存泄漏。

### 定时器轮盘

实现O(1)时间复杂度的非活跃连接检测与自动回收，有效降低系统开销。

### 灵活上下文机制

利用Any通用容器在连接生命周期中携带自定义上下文（如HttpContext），支持协议切换与状态保持。

### 缓冲区优化

自研环形缓冲区（Buffer），支持动态扩容与零拷贝数据移动，显著提升读写效率。

## 技术栈

`C++11` `Linux系统编程` `Epoll` `IO多路复用` `Reactor模型` `线程池` `状态机` `定时器轮盘` `RAII` `HTTP/1.1`

## 项目心得

这个项目是我对高并发网络编程的深度实践。从基础的多线程服务器到Reactor + Epoll架构，从阻塞IO到非阻塞事件驱动，每一步重构都让我对操作系统网络模型有了更深的理解。定时器轮盘实现O(1)超时检测、环形缓冲区减少内存拷贝，这些"底层优化"带来的性能提升让我意识到系统编程中每一个细节都至关重要。
